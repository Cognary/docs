# Overview

Aionis is execution memory infrastructure for production AI agents.

Aionis records what AI agents actually did and allows workflows to be replayed later.

## Why teams use Aionis

Most memory tools optimize only retrieval quality. Production teams also need:

1. Traceability: identify exactly which request, run, and decision changed behavior.
2. Control: apply rules before tools/actions are selected.
3. Replay: reconstruct and verify historical execution paths.
4. Operability: gate releases with repeatable checks.

## Key capabilities

1. Verifiable writes with `commit_id` and `commit_uri`.
2. Context assembly with explicit layer and budget controls.
3. Policy-loop endpoints for rule evaluation, selection, and feedback.
4. Decision provenance using `request_id`, `run_id`, `decision_id`, `commit_uri`.
5. Production gate workflow for safe release rollout.

## Who should read this

2. Platform engineers operating multi-tenant agent infrastructure.
3. AI product teams that need incident replay and decision auditability.
