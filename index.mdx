# Aionis

Aionis records what AI agents actually did and allows workflows to be replayed later.

Execution memory and replay infrastructure for production AI agents. Use Aionis when prompt traces and ad hoc logs are no longer enough to explain agent behavior in production.

## Who this documentation is for

This docs set is for teams shipping AI agents into real environments, where memory quality, tool decisions, incident debugging, and release safety all matter. It is written for developers, platform engineers, and operators who need to integrate Aionis and keep it reliable after launch.

## What Aionis gives you

1. Durable execution memory with explicit commit lineage.
2. Context assembly for bounded planner and runtime inputs.
3. Policy-aware tool selection with decision provenance.
4. Replay workflows for incident debugging, regression checks, and release evidence.

## Core operating model

1. Record what the agent did.
2. Recall and assemble the right context for the next step.
3. Apply policy before action execution.
4. Preserve the IDs needed to inspect or replay the workflow later.

1. Get a local or hosted environment responding to `/health`.
2. Write and recall a real memory event.
3. Run the policy flow with a stable `run_id`.
4. Keep the identifiers needed for replay and incident debugging.
5. Understand which section to use next for integration, operations, or troubleshooting.
